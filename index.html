📖 ETCC Silent Auction Manager — User Manual

SAM manages the full silent auction lifecycle for the East Tennessee Corvette Club — from item intake through payment and pickup. Follow the eight steps in order for each auction event.

Step 1 — Load Item Emails

Connects to etccwebsite.auction@gmail.com via Gmail (OAuth sign-in required) and scans the inbox for donation submission emails. Click Scan Inbox to authenticate and fetch emails, then Load Items to import new ones into the database. Duplicate emails are automatically skipped. Each item is assigned a unique number based on its category (e.g. 200-3 for the third Corvette item).

Step 2 — Create Bid Sheets

Generates a printable bid sheet for each item. Each sheet shows the item number, description, donor, value, starting bid, and a numbered table of bid rows with increment amounts. Starting bid and bid count are configurable in Settings. Select individual items or use Check All, then click Print.

Step 3 — Register Bidders

Add auction attendees as bidders and assign bidder numbers. Import members from a Club Express export file or a registration CSV. Walk-in bidder numbers start at the configured Walk-in Start value (default 1000). Each bidder record stores name, email, and phone.

Step 4 — Record Winning Bidders

After the auction closes, enter the winning bidder number and winning bid amount for each item. The table lists all items; type the bidder number and bid amount into each row and click Save. Only rows with both a bidder number and a bid amount are saved as winners.

Step 5 — Print Invoices

Generates printed invoices for winning bidders. Select individual bidders using the checkboxes or use Check All, then click Print All. Each invoice lists the bidder's winning items, amounts, total, and payment options including a credit card surcharge calculation based on the Square fee settings.

Step 6 — Announce Winners

Displays a table of all bidders who won one or more items. Use this screen to publicly announce winners at the event. The Email Bidders button opens the Email Bidders screen where you can send personalized winner notification emails.

Step 7 — Pay & Pickup

Records payment method and item pickup status for each winning bidder. Supports Cash, Check, Credit Card, and Other. Credit card totals automatically include the Square transaction fee (configurable in Settings). Mark each bidder as paid and items as picked up as they are processed.

Step 8 — Post Auction

Reporting and email tools for after the auction closes:

  • Auction Summary — full list of bidders, items, winners, and total income
  • Winning Bidder Report — each winner's items and amount due
  • Donor Report — items donated and revenue per donor
  • Club Financial Summary — income broken down by payment method
  • Email Bidders — send personalized winner emails or non-winner thank-you emails via Gmail API

Email Bidders

Opens a full-screen panel listing all bidders with their winner/non-winner status. Select the recipients using the checkboxes, then:

  • Send Winning Emails — sends a personalized email to each selected winner via the Gmail API. Each email lists only that winner's items and total. Gmail sign-in (Step 1) must be completed first.
  • Send NonWinners Emails — sends a single thank-you email to all selected non-winners as BCC.

Settings

  • Gmail OAuth Client ID — required for scanning the inbox and sending winner emails. Enter your Google Cloud OAuth 2.0 client ID.
  • Starting Bid % — starting bid as a percentage of item value (default 30%)
  • Final Bid % — highest bid row as a percentage of item value (default 150%)
  • Bid Count — number of bid rows on each bid sheet (default 25)
  • Walk-in Start — first bidder number assigned to walk-in registrations (default 1000)
  • Square Fee % and Square Flat Fee — credit card surcharge rates used on invoices and Pay & Pickup
  • Email Field Mapping — maps labels in donation emails to database fields. Adjust if your email format changes.
  • Data Backup — Export a full JSON backup of all auction data, or Import a previously saved backup to restore data.

Data & Sync

All data is stored in the browser's localStorage and automatically synced to a MySQL database on the server after every change. On startup, the app loads the latest data from the server so data is preserved across browsers and devices. Use Export Backup in Settings to save a local copy before major changes.
